Student Record API Collaboration Guide

This page explains exactly what each group member should do after cloning the project. The goal is for everyone to add one student record to the in-memory array in index.js and commit the change cleanly so it can be merged into the main project.

What to edit

  1. Open index.js.
  2. Find the in-memory students array.
  3. Add one new student object under the existing sample records.
  4. Use your own name, age, and course so your contribution is unique.

How to work

  1. Clone the repository.
  2. Create a branch for your work.
  3. Make only your own student record change.
  4. Run the app and confirm that GET /students shows your record.

How to commit

  1. Stage your changes.
  2. Write a clear commit message.
  3. Push your branch to GitHub.
  4. Open a pull request into main.
Recommended commit command

git add index.js
git commit -m "Add student record for Your Name"
git push origin your-branch-name

Important rule

  • Do not delete or overwrite someone else’s student record.
  • If the array changed since your clone, pull the latest changes first and then add yours.
  • Merge through a pull request so the main branch stays clean.

The API runs at http://localhost:3030 and this page is served from the public folder.
